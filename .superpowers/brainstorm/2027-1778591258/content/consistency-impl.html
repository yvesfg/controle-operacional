Seção 9 — Regras de Consistência (não quebrar)

🚫 Nunca fazer
✕Hardcodar cor (#f5c53a) em vez de var(--accent) — quebra o tema claro
✕Hardcodar z-index (9999) em vez de var(--z-modal) — empilhamento quebrará
✕Usar 'Space Grotesk' ou 'DM Mono' — fontes fora do sistema
✕Criar overlay com position:fixed inline — usar .co-modal-overlay
✕Radius diferente de 2/4/6/8/12px — fora da escala aprovada
✕Espaçamento fora da base 4px (ex: 7px, 11px, 15px)
✕Lógica de negócio dentro de componentes de UI (KpiCard, DataRow…)
✅ Sempre fazer
✓Toda cor via CSS var: var(--accent), var(--text), var(--card)
✓Toda fonte via token: var(--font-heading), var(--font-body), var(--font-mono)
✓Modais sempre via .co-modal-overlay + variante (--center, --top)
✓Animação de entrada: slideUp (center/top) ou mslide (bottom-sheet)
✓Novos modais → arquivo próprio em src/modals/Modal*.jsx
✓Props via ctx object (padrão já adotado em todos os modais)
✓StatusBadge para todo indicador de status — nunca inline ad hoc

StatusBadge — assinatura real
const STATUS_MAP = {
  'no-prazo':  { label:'No prazo', color:'var(--green)', bg:'rgba(14,203,129,.1)' },
  'aguardando': { label:'Aguardando', color:'var(--accent)', bg:'rgba(245,197,58,.1)' },
  'ro-pendente':{ label:'RO Pendente', color:'var(--red)', bg:'rgba(246,70,93,.1)' },
  'encerrado':  { label:'Encerrado', color:'var(--text2)', bg:'var(--card2)' },
};
export function StatusBadge({ status }) {
  const s = STATUS_MAP[status];
  return <span style={{background:s.bg, color:s.color, …}}>{s.label}</span>;
}
Tailwind — quando usar (e quando não)
✓ Usar Tailwind para:
Layout utilitário: flex gap-2 items-center
Responsividade: hidden md:flex, grid-cols-2 md:grid-cols-4
Estados hover: hover:opacity-80
✕ Não usar Tailwind para:
Cores: usar var(--accent), não text-yellow-400
Fontes: usar tokens, não classes Tailwind de font-family
Z-index: usar var(--z-modal), não z-50
